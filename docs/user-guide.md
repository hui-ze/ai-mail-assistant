## 1. 快速开始

### 1.1 安装与启动

**方法一：直接运行（推荐）**
- 解压下载包，双击 `Foxmail邮件助手.exe` 即可启动
- 无需安装，首次启动会自动引导完成配置

**方法二：源码运行（开发者）**
```bash
cd D:\ai-mail-assistant
python main.py
```

### 1.2 首次引导流程

首次启动时，程序会自动弹出三步引导向导：

| 步骤 | 内容 | 是否必填 |
|------|------|---------|
| 第1步：个人信息 | 姓名、部门、职位、工作描述 | 可跳过 |
| 第2步：配置邮箱 | 邮箱类型、地址、授权码 | 建议填写 |
| 第3步：AI 设置 | 本地 Ollama 或云端 API | 可跳过 |

> 💡 所有设置均可在「设置」页面随时修改

### 1.3 界面概览

```
┌──────────────┬─────────────────────────┬────────────────────┐
│  📥 邮件账号  │     邮件详情 & AI 摘要    │   ✅ 我的待办      │
│              │                         │                    │
│  账号列表     │  发件人 / 时间 / 主题     │  待办卡片          │
│  [+ 添加]    │                         │  [筛选] [同步日历]  │
│              │  【AI 摘要】             │                    │
│  邮件列表     │  本封邮件的智能摘要...    │                    │
│  [搜索] [刷新]│                         │                    │
│              │  【待办事项】            │                    │
│              │  ☐ 待办1                │                    │
│              │  ☑ 待办2（已完成）       │                    │
└──────────────┴─────────────────────────┴────────────────────┘
```

## 2. 配置邮箱账号

### 2.1 添加邮箱

1. 点击左侧面板下方「**+ 添加账号**」按钮
2. 在弹出对话框中：
   - 选择「邮箱类型」（QQ/163/Gmail 等）
   - 填写邮箱地址
   - 填写**授权码**（非登录密码，详见第7节）
3. 点击「保存」

### 2.2 支持的邮箱

| 邮箱类型 | IMAP 服务器 | 端口 |
|---------|-----------|------|
| QQ 邮箱 | imap.qq.com | 993 |
| 163 邮箱 | imap.163.com | 993 |
| 126 邮箱 | imap.126.com | 993 |
| Gmail | imap.gmail.com | 993 |
| Outlook | imap-mail.outlook.com | 993 |
| 腾讯企业邮 | imap.exmail.qq.com | 993 |

### 2.3 同步邮件

- **手动刷新**：点击邮件列表右上角「刷新」按钮
- **自动同步**：在「设置 → 通用」中配置同步间隔（默认 15 分钟）

## 3. AI 功能使用

### 3.1 AI 摘要生成

1. 在邮件列表中选中一封邮件
2. 点击中间面板下方「**AI 分析**」按钮
3. 等待 AI 处理（本地模型约 10-30 秒，云端 API 约 3-8 秒）
4. 查看生成的「摘要」和「待办事项」

### 3.2 AI 模型配置

**本地模型（推荐，隐私安全）**

需要先安装 Ollama：
```bash
# 1. 下载安装 Ollama: https://ollama.ai
# 2. 下载推荐模型
ollama pull qwen3:8b   # 约 5GB，中文效果好
# 3. 确保 Ollama 服务运行中
```

**云端 API**

在「设置 → AI 设置」中填入对应的 API Key：

| 服务商 | 获取地址 | 费用 |
|--------|---------|------|
| DeepSeek | platform.deepseek.com | 约 ¥0.001/千字 |
| 通义千问 | dashscope.aliyun.com | 约 ¥0.004/千字 |
| 文心一言 | cloud.baidu.com | 约 ¥0.005/千字 |

### 3.3 待办归属判断

AI 会自动判断邮件中的待办是「属于您」还是「需要他人完成」：

- **我的待办**：直接添加到您的待办列表
- **他人待办**：标记来源，可通过团队协作功能分配

## 4. 待办事项管理

### 4.1 查看待办

右侧「我的待办」面板展示所有待办，支持：
- **全部 / 未完成 / 已完成** 三种筛选
- 按优先级（高/中/低）显示
- 显示来源邮件

### 4.2 操作待办

| 操作 | 方法 |
|------|------|
| 标记完成 | 点击待办前的复选框 |
| 编辑内容 | 双击待办项 |
| 设置截止日期 | 右键 → 设置截止日期 |
| 同步到日历 | 点击待办卡片上的「同步日历」按钮 |
| 删除待办 | 右键 → 删除 |

### 4.3 同步到日历

支持三种日历：
- **本地日历（推荐）**：Windows 系统日历
- **Outlook 日历**：需配置 Outlook 账号
- **Google 日历**：需配置 Google API

在「设置 → 日历」中启用并配置。

## 5. 团队协作功能

### 5.1 部门管理

在「设置 → 部门协作」中：
1. 创建部门，设置共享文件夹路径
2. 添加团队成员（姓名、邮箱、角色）
3. 指定部门 Leader

### 5.2 待办同步

- **Leader**：可将待办上传到共享文件夹，成员自动获取
- **成员**：可从共享文件夹下载 Leader 分配的待办

### 5.3 待办分配

1. 在待办列表中选中一个待办
2. 右键 → 「分配给成员」
3. 选择成员，填写说明
4. 成员收到通知后可接受或拒绝

### 5.4 提醒功能

系统会在待办截止前自动提醒：
- **提前 1 天**：桌面通知
- **提前 1 小时**：桌面通知
- **提前 10 分钟**：紧急提醒

## 6. 常见问题解答

### Q1: 连接邮箱失败怎么办？

**检查清单：**
1. ✅ 确认邮箱开启了 IMAP 服务（不是 POP3）
2. ✅ 使用「授权码」而非登录密码
3. ✅ 检查网络连接
4. ✅ QQ 邮箱需要在设置中开启 IMAP 并生成授权码

### Q2: AI 摘要不准确怎么办？

- 切换到更强的模型（qwen3:8b → deepseek-v3 等）
- 云端 API 效果通常优于本地模型
- 邮件内容过短（< 50 字）时摘要效果有限

### Q3: Ollama 无法连接？

```bash
# 检查 Ollama 是否运行
ollama list
# 如果没在运行，启动它
ollama serve
```

确保地址填写为 `http://localhost:11434`

### Q4: 数据存储在哪里？

所有数据存储在本机，路径为：
```
C:\Users\{用户名}\.foxmail-assistant\
├── data.db     # 数据库（邮件、待办、设置）
└── logs\       # 运行日志
```

### Q5: 如何彻底清除数据？

删除 `C:\Users\{用户名}\.foxmail-assistant\` 整个文件夹即可。

## 7. 获取授权码指南

> ⚠️ **重要**：请使用「授权码」而非邮箱登录密码，更安全！

### QQ 邮箱

1. 登录 [mail.qq.com](https://mail.qq.com)
2. 点击右上角「设置」→「账户」
3. 找到「POP3/IMAP/SMTP/Exchange/CardDAV/CalDAV 服务」
4. 开启「IMAP/SMTP 服务」
5. 按提示发送短信验证
6. 复制生成的**授权码**（格式如：`abcdabcdabcdabcd`）

### 163 邮箱

1. 登录 [mail.163.com](https://mail.163.com)
2. 点击「设置」→「POP3/SMTP/IMAP」
3. 开启「IMAP/SMTP 服务」
4. 设置「客户端授权密码」（即授权码）

### Gmail

1. 访问 [myaccount.google.com](https://myaccount.google.com)
2. 「安全性」→「两步验证」（需先开启）
3. 「应用专用密码」→ 选择「邮件」→ 生成密码
4. 使用生成的 16 位密码作为授权码

### Outlook / Hotmail

- 使用账户的**登录密码**即可（Outlook 支持密码直接登录 IMAP）
- 若开启了二步验证，需要生成「应用密码」
